UNI

Violation of uniqueness

The UNI generator is used to detect uniqueness violations: i.e. instances, where an attribute in one table is not unique although it should be.

The UNI generator may be used directly as a worksheet function. Rather than doing this, we suggest using the UNI dialog from the menu bar, especially when using this function for the first time.

Consider the database of a retailer with two entities: purchases and customers. An attribute of each purchase is the purchase_id, which should serve as a unique identifier. Settings

The pattern is instantiated via four parameters. The table is defined by two parameters:

  • the schema
    (providing for databases where identically named tables may exist in different schemas)
  • the table

The third parameter specifies the attribute, that is supposed to be checked for uniqueness:

  • the unique key is an attribute of the table, that we would like to check for uniqueness

Composite keys can be chosen by separating every attribute with a comma. The fourth parameter defines the scope, i.e. the subset of the table to be included in the check:

  • the scope defines the subset of the referencing table to be included in the check

The scope is specified in the syntax of an SQL where clause. It is an optional parameter. When left blank, the scope is the entire table. Additional unique key settings

It is possible to check entries for uniqueness. So we are no longer looking for duplicate key entries, but rather for whole duplicate table entries. To achieve this, there are the following additional specifications for the parameter unique key.

  • * checks for duplicate table entries

When using either PostgreSQL, SQLServer, Oracle, or DB2, further commands can be used. To run a check on a list of possible composite keys, the input may look like this:

  • COL1, COL2, ... ? checks all combinations of the columns COLj
  • FIX1, FIX2, ... + SUP1, SUP2, ... checks all combinations of fixed columns FIXj and supplementary columns SUPj, that contain every fixed column. For example, FIX1, FIX2 + SUP1, SUP2 will check the combinations
    • FIX1, FIX2
    • FIX1, FIX2, SUP1
    • FIX1, FIX2, SUP2
    • FIX1, FIX2, SUP1, SUP2

Leaving out the supplementary columns lets UNI interpret every non-fixed column as supplementary. So if FIX1, FIX2, SUP1, SUP2 is the complete list of a table's columns, then the expression FIX1, FIX2 + would be a shortcut for FIX1, FIX2 + SUP1, SUP2.

  • *? checks every possible combination of attributes for duplicates

Execution

Once every compulsory field is filled, clicking the button Generate SQL Statement will create the corresponding query. To execute it, fill in the remaining output details (see [link: Control Panel User Guide]) and press the Execute Query button to generate the result set.
