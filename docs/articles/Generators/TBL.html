TBL

Summary of tables and schemas

The TBL generator is used to create an overview of the tables in a database or schema. It may be used directly as a worksheet function. Rather than doing this, we suggest using the TBL dialog from the menu bar, especially when using this function for the first time.


Figure 1: location of the TBL ribbon

Example

Consider the database of a retailer with two entities: purchases and customers. purchases consists of four columns and five rows while customers only has two of each. This is demonstrated in the below example.
Table 1: purchases Table 2: customers

The TBL generator creates an overview of the size of each table inside of this database. This overview contains:

  • TABLE_NAME, the name of each table
  • NUM_ROWS, the number of rows in the table
  • NUM_COLS, the number of columns in the table

Settings

The only setting needed to use the TBL generator is the selection of a database or schema. Make sure, that you have selected the correct connection in the Connection Overview panel (see [link: Connection Overview User Guide]). If TBL should only be applied to all tables in a schema of our database, we can select the lower option "All accessible tables in schema" and choose the schema.
