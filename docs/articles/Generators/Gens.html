SQL Generators

When working with large data sets, one often deals with the same recurring data quality issues. Therefore, RIVACON has developed 8 parameterizable, SQL-generating frameworks that tackle the core of these data quality problems and make your life easier. Every SQL Generator serves a certain purpose and outputs a SQL query, which - when executed - identifies a certain lack of data quality inherent in your data set. Additionally, each SQL Generator is available as an easy-to-use graphical user interface for an easier parametrization.

User interface

For each generator, the user interface is structured similarly. It consists of a Connection Overview, the generator's settings, a Query Editor, and the Control Panel (see Figure 1 below). For distinctive features of each generator, please refer to its main article on the left-hand side.

Figure 1: Generator structure

Connection Overview

This overview simplifies work with multiple data sources. The generated SQL query is always based on the database type of selected connection ID inside the drop-down menu in the top left corner. Please mind, that it might take a couple of seconds for the database structure to be loaded. Once loaded, the schemas and tables are listed in a "tree-view" structure to provide a better overview of your database.

Generator settings

As each generator creates a different type of SQL query, the selectable settings/parameters also differ for each generator. For detailed explanations and examples, please select the respective generator in the table of contents on the left-hand side. After filling the compulsory fields (marked by an asterisk), press the "Generate query" button.

Query Editor

To view the queries before executing them, they will appear in the Query Editor after pressing the "Generate query" button in the settings. More advanced users can also adjust the query when needed and will be assisted by syntax highlighting and text suggestions.

Control Panel

Inside the Control Panel, you can specify output properties before executing the SQL Query shown in the Editor. By default, a new worksheet will be created inside your Excel workbook with the top left corner of yout output matrix starting in cell A1. However, you may also select an already existing worksheet and another cell to be the top left corner of the output matrix. Please be aware, that by overwriting existing data with the output data of the executed query, you will not be able to revert the changes! Therefore, make sure to be careful when selecting an existing sheet as the output destination.

Furthermore, you have the option to leave out the table's column names, also known as the headers. "Clearing Old Contents" will clear all (old) data existing to the right and below your selected output cell.

Worksheet functions

Another way of using the generators is to call them as worksheet functions. Analog to popular Excel functions like SUM, IF or UNIQUE, simply typing the name of a generator behind an = sign will show the parameters (see Figure 2).

Figure 2: Generator Worksheet functions

Please note, by executing any SQL Generator as a worksheet function will only output the SQL query as a string but not execute the query itself. To execute the generated query, you'll need to wrap it with the SQL() function. For example: =SQL("YourConnectionID", DEF(...)).

Further information on the SQL() Worksheet function can be found in the chapter on "EasySQL".
