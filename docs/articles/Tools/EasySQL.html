EasySQL

Rather than using the =SQL(...) worksheet function inside a cell, EasySQL is the convenvient and user-friendly way to write and execute SQL queries with RIVALYTICS. It offers syntax highlighting and auto-completion while wirting queries as well as an overview of the selected connection in the "Connection Overview" section.


Figure 1: location of the EasySQL ribbon

Using EasySQL requires knowledge about SQL queries. Keep in mind, that the syntax must be adjusted based on the database type that is currently selected/active. This is demonstrated in the following example.

Oracle, PostgreSQL or IBM DB2 support "plain" SQL syntax with only minor differences between the three database management systems. For IBM DB2zOS and SQLServer, tables must be referenced between square brackets []. Additionally to the brackets, Excel demands the use of a $ sign after the table name, due to its support of custom, user-specified ranges. In the example above, by using [purchases$], the whole worksheet "purchases" is used as a data table, while e.g. [purchases$A1:C7] limits the data table range from A1 to C7.

Worksheet function

The execution of SQL statements can not only be done through EasySQL but also through the 'SQL()' worksheet function. Just like other common Excel worksheet functions like SUM, IF or UNIQUE, it can be easily executed from a cell behind an = sign. As in the graphical interface, it needs to be given a real or a dummy connection ID (see "Dummy Connections" in the chapter on the "Connection Manager") and a query.

Please make sure not to leave important information in the cells below and to the right of the cell executing the 'SQL()' function. They might be overwritten by the output table of 'SQL()', which will be inserted into your worksheet.
